# 翻译系统后台接口详细文档

## 1. 文档说明

本文档基于当前代码实现整理，覆盖 FastAPI 后端中已经实际注册的全部 `/api/v1/*` 接口，重点细化翻译系统相关接口，同时补充平台通用接口，便于：

- 前后端联调
- 第三方系统对接
- 测试用例编写
- 运维排障
- 二次开发

代码依据：

- `src/agent_system/api/__init__.py`
- `src/agent_system/models/__init__.py`
- `src/agent_system/services/translation_service.py`
- `src/agent_system/services/learning_service.py`
- `translate/models.py`

## 2. 基础约定

### 2.1 服务基础信息

- 基础路径：`/api/v1`
- 接口风格：REST + 少量 SSE 流式接口
- 数据格式：
  - 普通接口：`application/json`
  - 上传接口：`multipart/form-data`
  - 流式接口：`text/event-stream`
- 编码：`UTF-8`

### 2.2 认证方式

除以下接口外，其余 `/api/v1/*` 接口均要求登录认证：

- `GET /api/v1/health`
- `POST /api/v1/auth/login`

认证方式：

- 请求头：`Authorization: Bearer <access_token>`
- 也支持通过查询参数传递 `access_token`，流式接口常用

登录成功后返回：

- `access_token`
- `expires_at`
- `user`

### 2.3 统一响应与错误处理约定

接口没有统一包裹层，返回格式按业务对象决定，常见有三种：

- 直接返回单对象，例如 `Agent`、`Session`、`Skill`
- 返回数组，例如 `GET /api/v1/agents`
- 返回具名对象，例如 `{ "job": {...} }`、`{ "entries": [...] }`

常见错误码：

- `400`：参数错误、业务校验失败、文件格式不支持、配置未就绪
- `401`：未登录、Token 无效、Token 过期
- `404`：资源不存在
- `409`：资源冲突，例如任务运行中不允许删除、导出被质量问题阻断
- `422`：Pydantic 参数校验失败
- `429`：登录频率超限
- `502`：上游 LLM / 翻译后端异常

错误体典型格式：

```json
{
  "detail": "错误说明"
}
```

### 2.4 统一追踪头

所有请求都会带上或回传 `X-Trace-Id`：

- 请求头可主动传：`X-Trace-Id: <trace_id>`
- 服务端响应头一定返回：`X-Trace-Id`

用途：

- 串联日志
- 问题排查
- 跨接口调用追踪

#### 翻译作业状态 `JobStatus`

## 4. 健康检查与认证接口

### 4.1 `GET /api/v1/health`

用途：

- 检查服务是否可用
- 查看当前默认模型提供商和数据库路径

认证：

- 不需要

成功响应：

```json
{
  "status": "ok",
  "provider": "openai",
  "database_path": "/path/to/db.sqlite"
}
```

字段说明：

- `status`：固定为 `ok`
- `provider`：当前默认 LLM 提供商
- `database_path`：后端数据库文件路径

### 4.2 `POST /api/v1/auth/login`

用途：

- 登录后台系统并获取访问令牌

认证：

- 不需要

请求体：

```json
{
  "username": "admin",
  "password": "admin123"
}
```

参数说明：

- `username`：用户名，必填
- `password`：密码，必填

成功响应：

```json
{
  "access_token": "jwt-token",
  "expires_at": "2026-04-15T12:00:00+00:00",
  "user": {
    "id": "user_xxx",
    "username": "admin",
    "display_name": "System Admin",
    "role": "admin",
    "is_active": true,
    "created_at": "2026-04-15T11:00:00+00:00",
    "updated_at": "2026-04-15T11:00:00+00:00"
  }
}
```

错误：

- `401`：用户名或密码错误
- `429`：登录频率超限，同时响应头包含 `Retry-After`

补充说明：

- 默认引导账户由系统自动创建，默认用户名为 `admin`，默认密码为 `admin123`

### 4.3 `GET /api/v1/auth/me`

用途：

- 获取当前登录用户信息

认证：

- 需要 Bearer Token

成功响应：

- 返回 `User` 对象，字段与登录响应中的 `user` 一致

## 5. 翻译作业接口

这一组接口是翻译系统最核心的文档翻译 API。

### 5.1 关键对象：`TranslationJob`

翻译作业主对象字段如下：

- `id`：作业 ID，整数
- `source_file_name`：源文件名
- `source_file_path`：源文件绝对路径
- `target_file_path`：输出文件绝对路径
- `source_lang`：源语言，例如 `en`
- `target_lang`：目标语言，例如 `zh`
- `session_id`：关联会话 ID，可能为 `null`
- `task_run_id`：关联任务运行 ID，可能为 `null`
- `current_prompt_bundle_id`：当前提示词包 ID
- `source_format`：源文件格式，当前实现仅支持 `docx`、`pdf`
- `status`：作业状态，见 `JobStatus`
- `stage`：当前阶段，见 `JobStage`
- `page_count`：页面总数
- `total_segments`：总分段数
- `completed_segments`：已完成分段数
- `failed_segments`：失败分段数
- `progress_percent`：总体进度，0 到 100
- `document_summary`：文档分析摘要对象
- `translation_notes`：翻译说明数组
- `task_prompt_text`：实际使用的主翻译提示词
- `options`：作业创建选项
- `error_message`：错误信息
- `created_at`：创建时间
- `updated_at`：更新时间
- `started_at`：开始时间
- `completed_at`：完成时间
- `artifact_available`：译文文件是否已生成
- `quality_issue_counts`：质量问题计数，结构为 `{critical, warning, info}`
- `critical_issue_count`：严重问题数
- `warning_issue_count`：警告问题数
- `info_issue_count`：提示问题数
- `export_blocked`：是否禁止导出
- `export_status`：`ready` 或 `blocked`

### 5.4 `POST /api/v1/translations/upload`

用途：

- 上传源文档并创建翻译作业
- 服务端创建作业后会自动后台启动执行

认证：

- 需要

请求类型：

- `multipart/form-data`

表单参数：

- `file`：待翻译文件，必填，仅支持 `.docx`、`.pdf`
- `source_lang`：源语言，默认 `en`
- `target_lang`：目标语言，默认 `zh`

成功响应：

```json
{
  "job": {
    "id": 1,
    "source_file_name": "sample.docx",
    "status": "pending",
    "stage": "queued"
  }
}
```

错误：

- `400`：文件为空、格式不支持、翻译后端未配置、分析搜索未配置

注意：

- 当前实现要求在真实翻译后端模式下配置 `tavily_api_key`
- 若配置不满足，会返回：
  - `Translation backend is not configured...`
  - `Phase-1 translation analysis requires tavily_api_key...`

## 6. 术语学习接口

### 6.1 关键对象：`LearningJob`

字段说明：

- `id`：学习作业 ID
- `source_file_name`：源文文件名
- `source_file_path`：源文路径
- `translated_file_path`：译文路径
- `source_lang`：源语言
- `target_lang`：目标语言
- `project_id`：项目 ID，可空
- `domain`：领域
- `document_type`：文档类型
- `status`：作业状态
- `stage`：学习阶段
- `total_candidates`：候选术语总数

- `manual_changes`：人工干预次数
- `alignment_progress`：对齐阶段统计
- `stage_progress`：各阶段进度
- `stage_context_summary`：上下文摘要

### 6.9 `POST /api/v1/translation-learning/upload`

用途：

请求类型：

- `multipart/form-data`

- `domain`：领域，默认空字符串
- `document_type`：文档类型，默认空字符串
- `auto_materialize`：是否自动物化入库，默认 `true`

## 10. 对接建议

### 10.1 前端最常用接口组合

翻译工作台通常按以下顺序调用：

1. `POST /api/v1/auth/login`
2. `GET /api/v1/config`
3. `GET /api/v1/translations`
4. `POST /api/v1/translations/upload`
5. 轮询 `GET /api/v1/translations/{job_id}`
6. 成功后调用 `GET /api/v1/translations/{job_id}/artifact`

术语学习工作台通常按以下顺序调用：

1. `POST /api/v1/translation-learning/upload`
2. 轮询 `GET /api/v1/translation-learning/jobs/{job_id}`
3. 查看 `conflict_clusters` 与 `audit_artifacts`
4. 调用候选批准或驳回接口
5. 调用术语库与翻译记忆库查询接口校验落库结果

### 10.2 轮询建议

翻译作业和学习作业都采用后台异步执行，建议前端：

- 列表页轮询间隔：`5-10 秒`
- 详情页轮询间隔：`2-5 秒`
- 当状态进入 `completed | failed | cancelled | paused` 后停止高频轮询

### 10.3 高风险接口

以下接口建议在前端增加二次确认：

- `DELETE /api/v1/translations/{job_id}`
- `DELETE /api/v1/translation-learning/jobs/{job_id}`
- `DELETE /api/v1/translation-assets/glossary/{entry_id}`
- `DELETE /api/v1/translation-assets/tm/{entry_id}`
- `PUT /api/v1/config`

## 11. 备注

- 当前翻译上传接口仅支持 `DOCX` 和 `PDF`
- 当前学习上传接口支持 `DOCX`、`PDF`、`TXT`、`MD`
- 所有登录后接口默认要求 Bearer Token
- 流式接口返回的是 SSE 事件，不是标准 JSON 包
- 配置接口会返回敏感字段，例如 API Key；生产环境建议结合权限控制和脱敏策略
