# 前端端到端测试用例

## 1. 任务列表页面测试

### 1.1 页面加载与展示
**测试目标**: 验证任务列表页面正确加载和展示

**前置条件**: 
- 用户已登录系统
- 系统中存在多个翻译任务

**测试步骤**:
1. 访问任务列表页面 `/`
2. 等待页面加载完成

**预期结果**:
- 页面标题显示"翻译任务列表"
- 显示"创建任务"按钮
- 任务列表正确展示，包含任务名称、状态、进度、创建时间等信息
- 每个任务卡片显示操作按钮（查看详情、暂停/继续、取消、删除）

### 1.2 创建新任务
**测试目标**: 验证用户可以成功创建新的翻译任务

**前置条件**: 
- 用户在任务列表页面
- 准备好待上传的PDF文件

**测试步骤**:
1. 点击"创建任务"按钮
2. 在弹出的对话框中填写任务名称
3. 选择源语言（如：英语）
4. 选择目标语言（如：中文）
5. 上传PDF文件
6. 点击"创建"按钮

**预期结果**:
- 对话框关闭
- 显示成功提示消息
- 任务列表中新增一条任务记录
- 新任务状态为"待处理"或"处理中"

### 1.3 任务状态筛选
**测试目标**: 验证任务状态筛选功能

**前置条件**: 
- 系统中存在不同状态的任务（待处理、处理中、已完成、失败）

**测试步骤**:
1. 在任务列表页面，点击状态筛选下拉框
2. 选择"处理中"状态
3. 观察任务列表变化
4. 选择"全部"状态

**预期结果**:
- 选择"处理中"后，列表只显示处理中的任务
- 选择"全部"后，显示所有状态的任务
- 筛选过程流畅，无明显延迟

### 1.4 任务搜索
**测试目标**: 验证任务搜索功能

**前置条件**: 
- 系统中存在多个任务，包含已知名称的任务

**测试步骤**:
1. 在搜索框中输入任务名称关键词
2. 观察任务列表实时更新

**预期结果**:
- 列表只显示名称包含关键词的任务
- 搜索结果实时更新
- 清空搜索框后恢复显示所有任务

### 1.5 任务操作 - 暂停/继续
**测试目标**: 验证任务暂停和继续功能

**前置条件**: 
- 存在一个"处理中"状态的任务

**测试步骤**:
1. 找到处理中的任务
2. 点击"暂停"按钮
3. 观察任务状态变化
4. 点击"继续"按钮

**预期结果**:
- 点击暂停后，任务状态变为"已暂停"
- 按钮文字变为"继续"
- 点击继续后，任务状态恢复为"处理中"
- 显示相应的成功提示

### 1.6 任务操作 - 取消
**测试目标**: 验证任务取消功能

**前置条件**: 
- 存在一个"处理中"或"已暂停"状态的任务

**测试步骤**:
1. 找到目标任务
2. 点击"取消"按钮
3. 在确认对话框中点击"确定"

**预期结果**:
- 显示确认对话框
- 确认后任务状态变为"已取消"
- 显示成功提示
- 任务不再显示暂停/继续按钮

### 1.7 任务操作 - 删除
**测试目标**: 验证任务删除功能

**前置条件**: 
- 存在一个已完成或已取消的任务

**测试步骤**:
1. 找到目标任务
2. 点击"删除"按钮
3. 在确认对话框中点击"确定"

**预期结果**:
- 显示确认对话框，提示删除操作不可恢复
- 确认后任务从列表中移除
- 显示成功提示

### 1.8 分页功能
**测试目标**: 验证任务列表分页功能

**前置条件**: 
- 系统中存在超过10个任务

**测试步骤**:
1. 观察任务列表底部分页器
2. 点击"下一页"按钮
3. 点击具体页码
4. 修改每页显示数量

**预期结果**:
- 分页器正确显示总页数和当前页
- 点击翻页后正确加载对应页的任务
- 修改每页数量后列表正确更新

## 2. 翻译执行页面测试

### 2.1 页面加载与布局
**测试目标**: 验证翻译执行页面正确加载

**前置条件**: 
- 存在一个处理中的翻译任务

**测试步骤**:
1. 从任务列表点击"查看详情"进入翻译执行页面
2. 观察页面布局

**预期结果**:
- 页面分为左右两栏
- 左侧显示原文PDF预览
- 右侧显示翻译内容和执行日志
- 顶部显示任务信息（名称、状态、进度）
- 显示返回按钮

### 2.2 PDF预览功能
**测试目标**: 验证PDF预览功能正常工作

**前置条件**: 
- 在翻译执行页面

**测试步骤**:
1. 观察左侧PDF预览区域
2. 使用鼠标滚轮滚动PDF
3. 点击缩放按钮（放大/缩小）
4. 使用页码导航跳转到指定页

**预期结果**:
- PDF正确渲染显示
- 滚动流畅，无卡顿
- 缩放功能正常，内容清晰
- 页码导航准确跳转

### 2.3 实时翻译内容展示
**测试目标**: 验证翻译内容实时更新

**前置条件**: 
- 任务正在执行翻译

**测试步骤**:
1. 观察右侧翻译内容区域
2. 等待翻译内容逐步出现

**预期结果**:
- 翻译内容按页码和块ID组织展示
- 内容实时流式更新，无需刷新页面
- 显示当前翻译进度（如：第3页/共10页）
- 已完成的翻译块显示完整内容

### 2.4 执行日志展示
**测试目标**: 验证执行日志正确展示

**前置条件**: 
- 任务正在执行

**测试步骤**:
1. 切换到"执行日志"标签页
2. 观察日志内容

**预期结果**:
- 日志按时间顺序展示
- 显示各阶段执行状态（待处理、进行中、已完成、失败）
- 日志实时更新
- 不同状态使用不同颜色标识

### 2.5 质量问题展示
**测试目标**: 验证质量问题列表展示

**前置条件**: 
- 翻译过程中检测到质量问题

**测试步骤**:
1. 切换到"质量问题"标签页
2. 观察问题列表

**预期结果**:
- 显示问题类型、严重程度、位置、描述
- 问题按严重程度排序（严重、警告、提示）
- 点击问题可定位到对应的翻译内容

### 2.6 双向滚动同步
**测试目标**: 验证PDF预览和翻译内容的滚动同步

**前置条件**: 
- 在翻译执行页面，已有部分翻译内容

**测试步骤**:
1. 滚动左侧PDF预览
2. 观察右侧翻译内容是否同步滚动
3. 滚动右侧翻译内容
4. 观察左侧PDF是否同步滚动

**预期结果**:
- 滚动PDF时，翻译内容自动滚动到对应页码
- 滚动翻译内容时，PDF自动滚动到对应页面
- 同步流畅，无明显延迟

### 2.7 任务控制操作
**测试目标**: 验证在执行页面可以控制任务

**前置条件**: 
- 在翻译执行页面，任务处理中

**测试步骤**:
1. 点击"暂停"按钮
2. 观察任务状态和翻译更新
3. 点击"继续"按钮
4. 点击"取消"按钮并确认

**预期结果**:
- 暂停后翻译停止更新，状态变为"已暂停"
- 继续后翻译恢复更新
- 取消后任务停止，状态变为"已取消"
- 所有操作都有相应的提示反馈

### 2.8 任务完成处理
**测试目标**: 验证任务完成后的页面状态

**前置条件**: 
- 任务即将完成

**测试步骤**:
1. 等待任务完成
2. 观察页面变化

**预期结果**:
- 收到任务完成的SSE事件
- 进度显示100%
- 状态更新为"已完成"
- 显示成功完成的提示消息
- 控制按钮状态更新（隐藏暂停/继续按钮）

### 2.9 错误处理
**测试目标**: 验证任务失败时的错误处理

**前置条件**: 
- 模拟任务执行失败场景

**测试步骤**:
1. 触发任务失败（如网络中断、服务器错误）
2. 观察页面反应

**预期结果**:
- 显示错误提示消息
- 任务状态更新为"失败"
- 执行日志中记录错误信息
- 提供重试或返回列表的选项

## 8. 安全性测试

### 8.1 XSS防护
**测试目标**: 验证系统防止XSS攻击

**测试步骤**:
1. 在任务名称中输入 `<script>alert('XSS')</script>`
2. 创建任务
3. 观察任务列表显示

**预期结果**:
- 脚本代码被转义显示为纯文本
- 不执行JavaScript代码
- 页面正常显示

### 8.2 文件上传安全
**测试目标**: 验证文件上传的安全性

**测试步骤**:
1. 尝试上传非PDF文件（如.exe, .js）
2. 尝试上传超大文件（>100MB）

**预期结果**:
- 非PDF文件被拒绝，显示错误提示
- 超大文件被拒绝或限制
- 显示明确的文件要求说明

## 测试环境要求

- **浏览器**: Chrome 90+, Firefox 88+, Safari 14+, Edge 90+
- **屏幕分辨率**: 1920x1080, 1366x768, 768x1024, 375x667
- **网络环境**: 正常网络、慢速3G、离线
- **测试数据**: 准备不同大小的PDF文件（1页、10页、50页、100页）

## 测试工具建议

- **E2E测试框架**: Playwright 或 Cypress
- **性能监控**: Chrome DevTools Performance
- **网络模拟**: Chrome DevTools Network Throttling
- **跨浏览器测试**: BrowserStack 或 Sauce Labs
