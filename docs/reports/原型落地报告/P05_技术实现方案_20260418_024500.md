### 2.2 Mock API服务设计

#### 2.2.1 服务架构
```typescript
// src/services/mockApi.ts
class MockApiService {
  // 延迟模拟网络请求
  private delay(ms: number): Promise<void>
  
  // 任务相关
  getTasks(params): Promise<TaskListResponse>
  getTaskDetail(id): Promise<TaskDetailResponse>
  createTask(data): Promise<CreateTaskResponse>
  updateTask(id, data): Promise<UpdateTaskResponse>
  deleteTask(id): Promise<void>
  
  // 翻译段落相关
  getSegments(taskId, page): Promise<SegmentsResponse>
  updateSegment(taskId, segmentId, data): Promise<void>
  
  // 质量检查相关
  getQualityIssues(taskId): Promise<QualityIssuesResponse>
  resolveIssue(taskId, issueId): Promise<void>
  
  // 配置相关
  getLanguages(): Promise<Language[]>
  getGlossaries(): Promise<Glossary[]>
  getTranslationStyles(): Promise<TranslationStyle[]>
  
  // 统计相关
  getStatistics(): Promise<StatisticsResponse>
}
```

#### 2.2.2 响应格式统一
```typescript
interface ApiResponse<T> {
  code: number;           // 状态码（200成功，400/500错误）
  message: string;        // 响应消息
  data: T;                // 响应数据
  timestamp: string;      // 时间戳
}
```

### 2.3 数据完整性保证

#### 2.3.1 数据关联
- 任务ID关联：task-details、segments、quality-issues通过任务ID关联
- 配置ID关联：glossary_id、translation_style关联到config.json

#### 2.3.2 数据一致性
- 任务状态与段落进度一致
- 质量问题数量与任务统计一致
- 时间戳逻辑合理（created_at < started_at < completed_at）

#### 2.3.3 边界情况
- 空列表处理
- 不存在的ID返回404
- 参数验证错误返回400
