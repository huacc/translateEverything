const renderPageNumbers = () => {
  if (totalPages <= 7) {
    // 总页数<=7，显示所有页码
    return Array.from({ length: totalPages }, (_, i) => i + 1);
  }
  
  // 总页数>7，智能显示省略号
  if (currentPage <= 4) {
    return [1, 2, 3, 4, 5, '...', totalPages];
  } else if (currentPage >= totalPages - 3) {
    return [1, '...', totalPages - 4, totalPages - 3, totalPages - 2, totalPages - 1, totalPages];
  } else {
    return [1, '...', currentPage - 1, currentPage, currentPage + 1, '...', totalPages];
  }
};

## 🎯 技术亮点

### 1. 同步滚动算法优化

**问题**: 双向滚动同步容易导致循环触发，造成卡顿或死循环

**解决方案**:
- 使用`scrollingRef`标记当前滚动源
- 如果另一侧正在滚动，则忽略当前滚动事件
- 50ms后重置标志，允许下一次滚动

**效果**: 滚动流畅，无循环触发，用户体验良好

### 2. 页码导航智能显示

**问题**: 页数过多时，显示所有页码会导致导航栏过长

**解决方案**:
- 总页数<=7时，显示所有页码
- 总页数>7时，智能显示省略号
- 当前页居中显示，前后各显示1页
- 始终显示第1页和最后1页

**效果**: 导航栏简洁，易于操作，用户体验良好

### 3. 快捷键处理完善

**问题**: 快捷键可能与输入框冲突，导致误操作

**解决方案**:
- 监听全局键盘事件
- 检测焦点元素类型
- 如果焦点在输入框，则不响应快捷键

**效果**: 快捷键与输入框互不干扰，用户体验良好

### 4. 响应式设计

**问题**: 不同屏幕尺寸下，布局可能错乱

**解决方案**:
- 使用Flexbox布局
- 使用百分比宽度
- 使用媒体查询适配小屏幕

**效果**: 适配不同屏幕尺寸，布局美观
