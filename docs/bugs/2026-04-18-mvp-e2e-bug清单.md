# AI翻译系统 MVP 端到端 Bug 清单与修复记录
日期：2026-04-18

测试目标：
- 以前端需求文档为准，打通 MVP 主链路
- 使用 `backend_mock` 作为当前联调后端
- 覆盖任务列表、创建任务、翻译执行、对比审校、PDF 渲染与下载

测试环境：
- 前端：`http://127.0.0.1:4173`
- 后端：`http://127.0.0.1:8000`
- 后端运行环境：`backend_mock/.venv`

## 已修复

### 1. 创建任务页参数区空白
- 现象：后端 `/api/v1/config` 不可用时，第二步“配置参数”直接白屏，看起来像“参数没了”。
- 原因：前端只在加载失败时弹一次错误消息，没有加载态、错误态或回退配置。
- 修复：
  - 为创建任务页增加 `configLoading`、`configError` 和重试入口。
  - 为配置接口增加前端本地回退配置，避免参数区直接空白。
  - 恢复创建页、上传页、配置页、确认页的中文文案。

### 2. mock 后端配置接口和页面文案乱码
- 现象：参数文案、执行页文案、审校页文案出现乱码。
- 原因：部分前端文件和 `backend_mock` 返回内容被写成错误编码文本。
- 修复：
  - 重写 `backend_mock/main.py` 中的配置内容与 SSE 文案。
  - 重写前端任务创建、任务列表、翻译执行、审校页等核心页面文案。
  - 同步修复 `frontend/public/mock-data/config.json`。

### 3. 前端与 mock 后端接口协议不一致
- 现象：前端最初无法直接对接 `backend_mock`，任务列表、详情、上传、SSE 都不通。
- 修复：
  - `backend_mock` 已提供：
    - `GET /api/v1/config`
    - `GET /api/v1/translations`
    - `GET /api/v1/translations/{id}`
    - `POST /api/v1/translations/upload`
    - `POST /api/v1/translations/{id}/pause`
    - `POST /api/v1/translations/{id}/resume`
    - `POST /api/v1/translations/{id}/cancel`
    - `DELETE /api/v1/translations/{id}`
    - `GET /api/v1/translations/{id}/artifact`
    - `GET /api/v1/translations/{id}/stream`
    - `GET /api/v1/files/...`

### 4. 任务执行页存在 Hook 顺序风险与可视化不稳定
- 现象：旧版本执行页在热更新和任务状态切换时出现 Hook 顺序报错，SSE 与 PDF 状态容易错位。
- 修复：
  - 重写翻译执行页状态流转。
  - 保持 `useSSE`、`usePDFViewer` 的调用顺序稳定。
  - 统一暂停、继续、终止、完成后的前端状态刷新逻辑。

### 5. 审校页 PDF 渲染链路不稳
- 现象：原来审校页是占位结构，且缩放依赖 CSS 拉伸，容易让人误判为渲染异常。
- 修复：
  - 审校页改为左右真实 PDF 渲染。
  - 缩放改为直接使用 PDF.js 重绘，而不是先 1 倍渲染再用 CSS 缩放。
  - `PDFViewer` 增加高 DPI 画布处理，提升清晰度。

### 6. “译后 PDF 与真实 PDF 不一致”
- 现象：用户反馈页面显示的 PDF 内容与本地样本不一致。
- 根因：
  - 旧版 `backend_mock` 的种子文件不是 `样本` 目录中的真实年报。
  - 旧版 artifact 生成逻辑直接复制原文 PDF 作为“译文 PDF”。
- 修复：
  - `backend_mock` 现在直接使用 `样本/中文` 与 `样本/英文` 中的真实成对样本。
  - 已内置真实样本任务：
    - `AIA_2023_Annual_Report_zh.pdf -> AIA_2023_Annual_Report_en.pdf`
    - `AIA_2022_Annual_Report_zh.pdf -> AIA_2022_Annual_Report_en.pdf`
  - 上传的已知样本文件会按文件名自动匹配对应年份和目标语言的真实样本 PDF。

### 7. 用户本次截图涉及的“PDF 渲染错误”判定
- 结论：当前这次问题主要不是 PDF.js 把同一页渲染错了，而是比较对象不一致。
- 证据：
  - 页面截图中的页码是 `3 / 312`。
  - `312` 页对应当前任务 `AIA_2023_Annual_Report_zh.pdf`，不是 `AIA_2022_Annual_Report_zh.pdf`。
  - 当前审校页 `/tasks/3/review` 已验证为 `AIA_2022_Annual_Report_zh.pdf` 对 `AIA_2022_Annual_Report_en.pdf`，页码为 `3 / 304`。
  - 浏览器实测该页显示为目录页，和该真实样本第 3 页一致。

- `backend_mock/main.py` 语法校验通过
- 任务列表页可正常展示真实任务
- 创建任务页已恢复正常文案与上传入口，不再出现参数区白屏
- 翻译执行页可打开，PDF 可加载，执行日志与控制按钮可见
- 对比审校页可打开，左右 PDF 均为真实文件
- `GET /api/v1/translations` 已返回新的真实样本任务

## 当前仍存在的限制
- `backend_mock` 仍然是联调用 mock，不是真实翻译引擎。
- 对于不在 `AIA_20xx_Annual_Report_(zh|en).pdf` 规则内的任意上传文件，译后产物仍会退化为占位逻辑，无法保证真实双语对应。
- 执行页中的“翻译块摘要”目前是页级摘要或提示，不是逐块精确回填结果。
- 仍未实现真正的页内 bbox 级高保真回填。

## 下一步建议
- 把“上传任意样本 PDF -> 自动生成对应真实译后样本”范围从 AIA 年报扩展成更通用的样本映射表。
- 为执行页增加“当前加载原文文件路径 / 译文文件路径”的调试信息或开发态面板，避免再次把不同文件误判成渲染问题。
- 为审校页补充质量问题侧栏和差异说明，方便和需求文档中的审校体验对齐。
