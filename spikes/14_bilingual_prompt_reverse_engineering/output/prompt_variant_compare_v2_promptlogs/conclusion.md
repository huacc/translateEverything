# Prompt Compare Conclusion

## 结论

- 当前 4 套中，`baseline` 仍然是最稳的运行时提示词。
- `prompt_b` 接近 `baseline`，但更容易把正文往“商务文案化”方向推。
- `prompt_c` 没有跑赢 `baseline`，并且在 CFO 段出现了 `US$5.942 billion` 的单位错误。
- `prompt_a` 没有体现出稳定优势，更多是风格变重，但没有明显提高与人工译文的贴近度。

## 平均指标

- `baseline`: `avg_token_f1=0.5705`, `avg_sequence_ratio=0.2122`
- `prompt_b`: `avg_token_f1=0.5703`, `avg_sequence_ratio=0.2042`
- `prompt_c`: `avg_token_f1=0.5627`, `avg_sequence_ratio=0.2080`
- `prompt_a`: `avg_token_f1=0.5625`, `avg_sequence_ratio=0.1975`

## 应保留的提示词要素

- 明确语言方向：`中文 -> 英文`
- 明确场景：`港股/国际上市保险公司年度报告`
- 明确核心约束：`只翻译，不解释，不增删，不自由改写`
- 明确准确性要求：`数字、货币、百分比、年份、专有名词必须准确`
- 明确术语要求：`优先使用保险/金融/精算/治理固定译法`

## 建议删除或弱化的要素

- 夸张专家人设：如 `国际顶级`、`20年经验`、`四大背景`
- 抽象口号式约束：如 `先保真，再表达`
- 容易诱导改写的指令：如 `句式重构`、`逻辑对齐`
- 与运行时无关的流程描述：如分析步骤、自检步骤、清单输出要求

## 建议

- 生产链路先继续使用 `baseline`。
- 如果后续继续调优，不要再加长人设，而是只增加小而硬的约束。
- 下一轮更值得测的是：
  - 小规模术语表注入
  - 单位与数字校验后处理
  - 页面/章节级背景，而不是更长的人设提示词
